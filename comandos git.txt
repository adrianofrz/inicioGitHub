/**************************/
git init   -- inicializa um repositório e fica enxergando as mudanças dentro do projeto
git status -- mostra os status dos arquivos que fazem parte daquele repositório (untracked, unmodifed, modified e staged)
git add    -- adiciona um arquivo novo ao repositório local ou aplica as modificações realizadas no arquivo para que ele fique no estado de stage, logo ao realizar um commit esse arquivo será levado junto na criação da versão.
git commit -m "sua mensagem aqui" ou -am "sua mensagem aqui" (Esse aqui commita arquivos que já foram adicionados anteriormente ao repositório)-- pega todos os arquivos em modo stage (prontos para commit) e cria uma versão dele, um snapshot.
git log --visualiza as modificações realiazdas no repositório (parametros para o git log: shortlog mostra os arquivos que um determinado autor alterou e commitou
log --author="nome do cabrunco aqui": filtra os commits desse usuário
log --decorate: mostra de onde veio o arquivo e para qual branch ele foi
log --graph: mostra de forma gráfica o que foi realizado)
git show <hash do commit aqui> : mostra o que foi alterado no determinado arquivo
git diff -- mostra a diferença dos arquivos atuais com os últimos commits deles
git checkout "seu arquivo aqui" -- comando que desfaz uma modificação antes de ser adicionada
git reset HEAD "comandos git.txt" -- remove o arquivo do status stage, ou seja, pronto para ser commitado
git reset --soft ou --mixed ou --hard <hash do commit aqui>: Esses comando escolhem o commit que vão olhar, logo tudo o que foi feito depois deles, serão feitas as seguintes modificações 
--soft: apaga o commit realizado no repositório, volta os mesmos para o estado de stage para serem commitados de novo
--mixed: apaga o commit realizado no repositório, volta os mesmos para o estado de modificado para serem adicionados de novo
--hard: apaga o commit realizado no repositório e as modificações do arquivo
